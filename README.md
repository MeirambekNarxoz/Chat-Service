# Chat-Service

Микросервис для обмена сообщениями в реальном времени, хранения истории чатов и загрузки медиафайлов.

## Функциональные возможности
- **Персональные чаты**: Создание чатов 1-в-1.
- **История сообщений**: Получение истории любого чата из БД Postgres.
- **Медиа-файлы**: Загрузка картинок и документов в хранилище MinIO.
- **Real-time Messaging**: Мгновенный обмен сообщениями через WebSockets.
- **Global Chat**: Специальный режим "Общего чата" (через ChatID: 0).
- **Heartbeat**: Поддержка стабильного соединения (Ping/Pong).

---

## Режимы работы чата

### 1. Общий чат (Для всех)
Для отправки сообщения всем пользователям, которые сейчас находятся в сети:
- Используйте `chat_id: 0` в JSON-сообщении через WebSocket.
- Эти сообщения **не сохраняются** в базу данных (используются для быстрого уведомления всех онлайн-пользователей).

### 2. Приватные чаты (1 на 1)
Для безопасного общения между конкретными людьми:
- Сначала создайте чат через `POST /api/chat/create`.
- Используйте полученный `chat_id` (больше 0) в WebSocket.
- Сообщения **автоматически сохраняются** в Postgres и доставляются только участникам этого чата.

## REST API (Эндпоинты через Gateway :8080)

| Метод | Путь | Описание |
| :--- | :--- | :--- |
| `POST` | `/api/chat/create` | Создать чат (нужен `recipient_id`) |
| `POST` | `/api/chat/group` | Создать групповой чат (нужны `name`, `user_ids`) |
| `GET` | `/api/chat/list` | Список чатов текущего пользователя |
| `GET` | `/api/chat/history/:id` | Загрузить историю сообщений чата |
| `POST` | `/api/chat/messages/upload` | Загрузить файл в MinIO (Bucket: `image`) |

## WebSocket (Живое общение)

**Адрес для подключения**: `ws://localhost:8080/api/chat/ws?token=YOUR_JWT`

**Формат сообщения (JSON)**:
```json
{
  "chat_id": 1,
  "text": "Привет, это приватное сообщение!",
  "file_url": "" 
}
```
*(Для общего чата используйте `"chat_id": 0`)*.

## Настройка и запуск
1. Отредактируйте `.env`:
   ```env
   PORT=8085
   DB_URL=host=localhost user=postgres password=postgres dbname=postgres port=5432 sslmode=disable
   MINIO_ENDPOINT=localhost:9000
   MINIO_ACCESS_KEY=minioadmin
   MINIO_SECRET_KEY=minioadmin
   JWT_SECRET=your_jwt_secret
   ```
2. Запустите: `go run cmd/server/main.go`
