# Chat-Service (Premium 1-на-1)

Этот микросервис обеспечивает современный и быстрый опыт общения 1-на-1, поддерживая все функции, необходимые для продвинутого UI (как в SkillHub).

## Запуск

1. Поднять инфраструктуру из корня монорепо:
   ```bash
   docker compose up -d chat-db minio
   ```
2. Скопировать/проверить `.env` (порт БД `5441`, JWT_SECRET как у User Service).
3. Запуск **из папки `chat-service`**:
   ```bash
   go run ./cmd/server
   ```
4. REST идёт через API Gateway `http://localhost:8080/api/chat/...`, WebSocket — `ws://localhost:8080/api/chat/ws?token=JWT`.

> MinIO не обязателен для старта: без него чат работает, загрузка файлов отключена.

## Основные возможности
- **Rich Chat List**: Получение списка чатов с последним сообщением и счетчиком непрочитанных.
- **Read Status**: Поддержка статусов "Прочитано" для каждого сообщения.
- **WebSocket**: Мгновенная доставка сообщений и обновление статусов.
- **MinIO**: Загрузка и хранение медиафайлов.
- **Global Chat**: Специальный режим вещания (ChatID: 0) для тестов.

## REST API (Основные эндпоинты)

### Список чатов (диалогов)
**GET** `/api/chat/list`
Возвращает "обогащенный" список для главной страницы сообщений.
```json
[
  {
    "chat_id": 1,
    "recipient_id": 10,
    "last_message": {
        "text": "Привет! Как дела?",
        "created_at": "..."
    },
    "unread_count": 5
  }
]
```

### Пометка прочтения
**POST** `/api/chat/read/:chat_id`
Сбрасывает счетчик непрочитанных сообщений в данном чате.

### История сообщений
**GET** `/api/chat/history/:chat_id`
Возвращает все сообщения чата (включая статус `is_read`).

## WebSocket (Real-time)

**Подключение**: `ws://localhost:8080/api/chat/ws?token=...`

**JSON формат**:
```json
{
  "chat_id": 1,
  "text": "Привет!",
  "file_url": ""
}
```
